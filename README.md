# MY-TRADING-JOURNAL
BemoEdge is a trading performance operating system designed to help traders track trades, manage capital, monitor payouts, and build discipline. It combines analytics, journaling, financial management, and account dashboards into one unified platform for consistent, data-driven trading growth.

# BemoEdge – Trading Performance App

## Overview

BemoEdge is a full stack web app I built to help traders track their performance and manage their money properly.

The idea came from seeing how most traders don’t have structure. They don’t track properly, they don’t manage capital well, and they don’t treat trading like a business.

So I decided to build a system that brings everything into one place.

---

## What the app does

The platform allows traders to:

* Log and track trades
* View trade history and edit or delete trades
* See daily results using a trading calendar
* Track multiple accounts (funded and personal)
* Manage capital and balances
* Track payouts and withdrawals
* Track external income and expenses
* View total wealth and performance
* Write and store journal entries
* Set trading rules for discipline

Everything is connected so changes in one place affect the rest of the system.

---

## Tech Stack

### Frontend

* Next.js (App Router)
* React
* Tailwind CSS
* Framer Motion (for animations)

### Backend

* Supabase (PostgreSQL database + authentication + realtime)
* Next.js API routes for backend logic

### Deployment

* Vercel (frontend hosting)
* Supabase cloud (backend)

## How I built it

I started by building the trade tracker and dashboard. After that, I added more features step by step.

First:

* trade logging
* basic dashboard

Then I added:

* analytics
* trading calendar
* capital management

After that, I expanded the system into a full financial tracker:

* payouts
* expenses
* external income
* wealth tracking

I also worked on:

* UI design (dark theme, clean layout)
* sidebar navigation
* smooth animations

The goal was to make it feel like a real trading system, not just a simple app.

---

## System Structure

The app is built around a few main parts:

* Trades (for trading activity)
* Accounts (for capital and balances)
* Financial transactions (income, expenses, payouts)

These are connected so the dashboard always shows updated totals.
